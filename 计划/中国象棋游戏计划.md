# 中国象棋游戏 — 实施计划

## 背景

在 photo-manager-v2 中内置中国象棋游戏，作为游戏中心第二个可玩游戏。两人可在同一设备上对弈（pass-and-play），后续可扩展联机对战。

## 开源参考

| 库 | 用途 | 许可证 | 文件大小 | 问题 |
|---|------|--------|----------|------|
| [xiangqi.js](https://github.com/lengyanyu258/xiangqi.js) | 游戏逻辑（走法生成、将军检测、FEN） | MIT | ~15KB 单文件 | 仅 GitHub，未发布 npm |
| [xiangqiground](https://github.com/lucaferranti/xiangqiground) | 棋盘 UI（SVG 渲染、拖拽） | **GPL-3.0** | ~10KB gzipped | 传染性许可证；ESM only，需打包 |
| [elephantops](https://www.npmjs.com/package/elephantops) | 游戏逻辑（TypeScript） | MIT | ~30KB | 依赖多，需打包 |

**方案选择：**

- **游戏逻辑** → 参考 xiangqi.js（MIT），内联实现核心规则模块，不直接复制代码，学习其算法思路后自写
- **棋盘渲染** → Canvas 自绘，项目已有拼贴墙 Canvas 经验
- **不选 xiangqiground 的原因**：GPL-3.0 强制整个项目开源为 GPL，与项目现状冲突

## 架构

```
games/chinese-chess.js    -- 单文件，包含全部游戏逻辑 + Canvas 渲染
  ├── 棋盘模型层：局面表示、走法生成、将军/将死检测
  ├── Canvas 渲染层：棋盘绘制、棋子绘制、选高亮
  └── 交互层：点击选子/走子、走法历史、悔棋
```

注册到 GameEngine：`GameEngine.registerGame('chineseChess', new ChineseChessGame())`

## 棋盘模型（10×9 中国象棋标准棋盘）

### 棋子类型与编码

| 红方 | 黑方 | 名称 | 红方编码 | 黑方编码 |
|------|------|------|----------|----------|
| 帅 | 将 | King | K | k |
| 仕 | 士 | Advisor | A | a |
| 相 | 象 | Bishop | B | b |
| 馬 | 馬 | Knight | N | n |
| 車 | 車 | Rook | R | r |
| 炮 | 砲 | Cannon | C | c |
| 兵 | 卒 | Pawn | P | p |

### 局面表示

使用 FEN 字符串（中国象棋变体）作为内部 state，与主流库兼容：

```
初始局面 FEN:
rnbakabnr/9/1c5c1/p1p1p1p1p/9/9/P1P1P1P1P/1C5C1/9/RNBAKABNR w
```

### 核心规则实现

1. **基础走法** — 7 种棋子的合法目标格生成
2. **蹩马脚** — 马腿位置检测
3. **塞象眼** — 象心位置检测
4. **炮架** — 炮的翻山吃子条件
5. **将帅对面** — flying general 规则
6. **将军检测** — 走子后己方是否被将
7. **将死/困毙检测** — 无合法走法
8. **长将/长捉检测** — 简化版（三次重复局面判和）

## Canvas 渲染设计

### 棋盘布局

- 棋盘尺寸自适应容器宽度，保持 9:10 比例
- 网格线 + 楚河汉界标注
- 九宫格斜线
- 坐标标注（红方在下，黑方在上）

### 棋子渲染

- 圆形棋子：外圈 + 内圈 + 文字
- 红方：浅底红字；黑方：浅底黑/绿字
- 选中态：金色外圈高亮
- 可选走法提示：半透明圆点

### 交互

- 点击棋子 → 选中，显示合法走法提示
- 点击目标格 → 执行走子
- 吃掉对方棋子：显示在棋盘旁"已吃子"区域
- 悔棋按钮：撤销上一步
- 新局按钮：重置棋盘

## 用户界面

### 游戏内 HUD

```
[红方] 老大    VS    [黑方] 小弟
步数: 12    当前: 红方走棋
[悔棋] [新局] [翻转棋盘]
```

### 游戏模式

**Phase 1：本地对弈**
- 同一设备两人轮流走子（pass-and-play）
- 双方均使用同一登录用户的视角
- 红方先走，轮流进行

**Phase 2（后续）：联机对弈**
- 利用 Supabase Realtime 同步走子
- 挑战模式：一方发起，另一方接受
- 走子记录存 `game_scores.extra_data`

## 游戏计分规则

| 结果 | 胜方得分 | 负方得分 |
|------|----------|----------|
| 将死 | 100 | 0 |
| 对方认输 | 100 | 0 |
| 困毙 | 100 | 0 |
| 和棋 | 50 | 50 |

步数越少得分越高（bonus = max(0, 50 - 步数)），鼓励速胜。

## RPG XP 奖励

| 行为 | XP |
|------|-----|
| 完成一局（胜） | 20 |
| 完成一局（负/和） | 10 |
| 每日任务"下一局象棋" | +15 |

## 验证方式

1. 桌面版：游戏中心 → 点击中国象棋 → 棋盘渲染正确 → 可点击走子 → 将军提示 → 结束计分
2. 移动版：同上，触摸操作正常
3. 规则验证：测试马脚、象眼、炮架、将帅对面等边界情况
4. 排行榜：记录存入 game_scores (game_name = 'chinese_chess')
