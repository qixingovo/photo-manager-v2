# 经期忌口打卡 — 设计文档

**日期**: 2026-05-12  
**状态**: 已确认  
**平台**: 仅移动端

---

## 概述

在经期窗口期（预测经期前 3 天 → 经期结束），触发每月特殊忌口打卡任务。每日打卡确认忌口完成，获得 XP + 称号奖励。与现有 RPG 成就系统和经期追踪联动。

---

## 数据模型

### 新表：`dietary_checkins`

| 字段 | 类型 | 说明 |
|------|------|------|
| `user_name` | text | 用户标识 |
| `checkin_date` | date | 打卡日期 |
| `completed` | boolean | 当日是否完成忌口，默认 true |
| `note` | text (nullable) | 可选备注 |

唯一约束：`(user_name, checkin_date)`

### 扩展：`rpg_progress` 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `dietary_month_count` | integer | 累计完成的月度忌口次数，默认 0 |
| `dietary_custom_rewards` | jsonb | 自定义奖励数组，默认 [] |

### 忌口清单（代码常量）

```
冰的 / 辣的 / 咖啡 / 酒 / 生冷 / 油炸 / 奶茶
```

### 忌口窗口判定

复用 `period_daily_records` 的预测逻辑：
- 起始：`_getPredictedPeriodStart()` - 3 天
- 结束：当月最后一个 `is_period=true` 的实际日期
- 无预测数据时不触发

---

## RPG 联动规则

### XP
- 每次忌口打卡 +30 XP（计入每日 XP 上限 200）

### 月度完成
- 忌口窗口内**每天都有打卡记录** → 本月忌口挑战完成
- `dietary_month_count += 1`
- 新窗口开启时自动结算上月

### 称号
| 累计月度完成 | 称号 |
|-------------|------|
| 1 次 | 好宝宝 |
| 3 次 | 乖宝宝 |
| 6 次 | 好乖宝宝 |

### 自定义奖励
- 复用 `addCustomReward()`，关联忌口目标
- 格式：`{ type: 'dietary', target_month: N, reward: '...' }`

## UI 设计

### 首页浮球
- 仅忌口窗口期内显示
- 位置：首页右侧，垂直居中，不占 3×2 卡片布局
- 外观：粉色渐变圆球（52×52px），内含 🍽️ + "忌口"文字
- 待打卡时显示红点标记
- 点击弹出打卡弹窗

### 经期追踪页嵌入
- 位置：周期日历下方
- 仅忌口窗口期内显示
- 内容：窗口进度（第 X 天/共 Y 天）、忌口清单标签、打卡按钮/已完成状态

### 打卡弹窗
- 显示当天日期 + 经期第几天
- 展示忌口清单标签（不可交互，仅展示）
- 可选备注输入框
- "算了" / "完成打卡 +30 XP" 两个按钮

## 打卡流程

```
经期预测前3天 → 忌口窗口开启
  → 首页右侧出现 🍽️ 圆球（带红点）
  → 经期追踪页出现忌口卡片
  → 用户点击 → 弹出打卡弹窗
  → 点击"完成打卡" → +30 XP
  → 圆球变已打卡状态，当天不再提醒
  → 窗口结束 → 圆球消失
  → 判定：窗口内每天都有打卡？
    ✓ → 月度完成 +1，检查称号升级
    ✗ → 不计入月度完成
```

## 边界情况

| 情况 | 处理 |
|------|------|
| 经期未记录 / 无预测数据 | 忌口窗口不触发 |
| 漏打卡 | 不支持补卡，当天未打即失效 |
| 窗口重叠（周期 < 21 天） | 按每次经期窗口独立判定，一个窗口结束后结算，dietary_month_count 跨窗口累加 |
| 仅移动端 | 桌面端不实现此功能 |

---

## 实现方案

方案 A：扩展现有 RPG 任务系统。在每日任务机制上新增"经期忌口"特殊任务类型，忌口窗口内强制作为每日必选任务。打卡、XP、称号都走现有 RPG 管道。

### 未来扩展（方案 C）
通用"条件触发事件"子系统：经期是触发器的一种，忌口打卡是一种事件类型。可扩展到节假日特殊任务、纪念日任务等。当前仅一个用例，暂不实现。
