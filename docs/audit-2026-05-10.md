## Supabase SQL Editor — 直接复制粘贴执行

如果数据库已运行，只需创建缺失的策略：

```sql
-- ============================================
-- 1. app_settings anon SELECT（如果尚未创建）
-- ============================================
CREATE POLICY "app_settings_select_anon" ON app_settings
    FOR SELECT TO anon USING (true);

-- ============================================
-- 2. 刷新 PostgREST schema cache
-- ============================================
NOTIFY pgrst, 'reload schema';
```

> 注意：此策略已包含在 `migrations/011_app_settings_anon_select.sql` 中，重建数据库时按序执行即可。
