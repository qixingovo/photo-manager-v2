## 生产部署教训

本次安全加固后生产部署遇到的核心问题及教训：

1. **CDN 库行为差异** — supabase-js CDN 版（2.101.1）的 localStorage session 自动持久化在某些环境不生效，需要手动 save/restore 兜底。升级到 npm 包版本可规避。

2. **列类型不匹配** — `milestones.category_id`(BIGINT) 与 `categories.id`(UUID) 类型不兼容，导致引用完整性被破坏。未来新增外键引用应先确认双方列类型一致。

3. **RLS 收紧的连锁反应** — `app_settings` 的表在登录前被查询（`anniversary_start_date`），RLS 收紧后 406。对于登录前必需的数据，需要保留 anon SELECT。

4. **桌面版/移动版代码分叉** — 修复 `app.js` 后遗漏了 `mobile-app.js` 的同步修改，导致移动端仍使用旧逻辑。两端一致性问题需要纳入审核检查项。
